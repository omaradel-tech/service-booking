# Subscription Platform API

A production-grade Laravel 13 API application built with clean architecture for managing service subscriptions, bookings, and cart functionality.

## Features

- **User Authentication**: Registration, login, and API token management using Laravel Sanctum
- **Service Management**: Browse and manage maintenance services
- **Booking System**: Schedule and manage service appointments with overlap prevention
- **Shopping Cart**: Add services and packages to cart with quantity management
- **Package Management**: Service bundles with automatic discount calculation
- **Subscription Management**: Trial and paid subscriptions with grace periods
- **Clean Architecture**: Domain-driven design with separation of concerns
- **Comprehensive Testing**: Feature and unit tests using Pest
- **API Documentation**: Auto-generated documentation with Scribe
- **Logging & Monitoring**: Multi-channel logging with Log Viewer
- **Caching**: Redis-based caching for performance optimization

## Architecture

This application follows clean architecture principles with clear separation of concerns:

```
app/
|-- Core/
|   |-- Domain/           # Business logic and enums
|   |-- Application/      # Services, DTOs, and interfaces
|   |-- Infrastructure/   # Repository implementations
|-- Modules/
|   |-- User/            # Authentication and user management
|   |-- Service/         # Service catalog
|   |-- Booking/         # Appointment booking
|   |-- Cart/            # Shopping cart functionality
|   |-- Package/         # Service bundles
|   |-- Subscription/    # Subscription management
```

## Installation

1. **Clone the repository**
   ```bash
   git clone <repository-url>
   cd subscription
   ```

2. **Install dependencies**
   ```bash
   composer install
   ```

3. **Environment setup**
   ```bash
   cp .env.example .env
   php artisan key:generate
   ```

4. **Configure environment variables**
   ```env
   DB_CONNECTION=pgsql
   DB_HOST=127.0.0.1
   DB_PORT=5432
   DB_DATABASE=subscription
   DB_USERNAME=your_username
   DB_PASSWORD=your_password

   CACHE_STORE=redis
   REDIS_HOST=127.0.0.1
   REDIS_PASSWORD=null
   REDIS_PORT=6379
   ```

5. **Run migrations**
   ```bash
   php artisan migrate
   ```

6. **Seed the database**
   ```bash
   php artisan db:seed
   ```

7. **Start the development server**
   ```bash
   php artisan serve
   ```

## API Documentation

Once the application is running, visit `http://localhost/docs` to view the comprehensive API documentation generated by Scribe.

### Authentication

All protected endpoints require a Bearer token obtained through the login endpoint:

```bash
curl -X POST http://localhost/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email": "user@example.com", "password": "password"}'
```

Include the token in subsequent requests:

```bash
curl -X GET http://localhost/api/v1/services \
  -H "Authorization: Bearer {your-token}"
```

## Key Endpoints

### Authentication
- `POST /api/v1/auth/register` - Register new user
- `POST /api/v1/auth/login` - User login
- `POST /api/v1/auth/logout` - User logout
- `GET /api/v1/auth/me` - Get current user profile

### Services
- `GET /api/v1/services` - List active services
- `GET /api/v1/services/{id}` - Get service details

### Bookings
- `GET /api/v1/bookings` - List user bookings
- `POST /api/v1/bookings` - Create new booking
- `GET /api/v1/bookings/{id}` - Get booking details
- `POST /api/v1/bookings/{id}/cancel` - Cancel booking

### Cart
- `GET /api/v1/cart` - Get user cart
- `POST /api/v1/cart/items` - Add item to cart
- `PUT /api/v1/cart/items/{id}` - Update cart item
- `DELETE /api/v1/cart/items/{id}` - Remove item from cart

### Packages
- `GET /api/v1/packages` - List active packages
- `GET /api/v1/packages/{id}` - Get package details

### Subscriptions
- `GET /api/v1/subscriptions/current` - Get current subscription
- `POST /api/v1/subscriptions/cancel` - Cancel subscription
- `GET /api/v1/subscriptions/check` - Check subscription status

## Business Logic

### Subscription Management
- Users get a 30-day trial subscription upon registration
- Trial subscriptions include a 7-day grace period
- Active subscriptions are required for creating bookings
- Subscriptions automatically expire after the end date

### Booking Rules
- Bookings require an active subscription
- Booking times cannot overlap for the same user
- Bookings can only be canceled if they're pending/confirmed and in the future
- Booking confirmation emails are sent automatically

### Cart Management
- Users can add services and packages to cart
- Cart items have quantity limits (max 10)
- Cart totals are calculated automatically
- Cart items are removed when bookings are created

### Package Pricing
- Packages must be priced lower than the sum of their services
- Discount percentages are calculated automatically
- Inactive services cannot be added to packages
